# 三星堆交互系统用户使用文档

## 1. 项目简介

本项目是一个基于网页的三星堆文物粒子交互展示系统。用户可以在浏览器中查看星云粒子效果、导入 `.obj` 文物模型、切换文物、删除文物、调整粒子颜色、使用鼠标/滚轮/手势进行交互，并通过 API 自动生成文物描述。

## 2. 文件说明

下载并解压压缩包后，通常会看到以下内容：

- `index.html`：主网页文件，双击或通过本地服务器打开即可运行。
- `assets/models/`：预设文物模型文件夹，里面可放置 `.obj` 文物模型。
- `.vscode/`：VS Code/Cursor 的本地调试配置文件夹，用于一键启动本地服务器并自动打开网页。
  - `launch.json`：负责启动浏览器调试页面。
  - `tasks.json`：负责在运行前自动启动 Python 本地服务器。
- `用户使用文档.md`：当前使用说明文档。

请不要随意移动 `assets` 文件夹，否则预设文物模型可能无法加载。

## 3. 新用户下载压缩包后的打开方式

### 推荐方式：使用本地服务器打开

由于本系统包含 `.obj` 模型预加载、摄像头手势识别、外部脚本库和 API 请求，建议不要直接双击 `index.html` 打开，而是使用本地服务器运行。

### Windows 用户打开方式

#### 方法一：使用 Python 本地服务器（Windows 推荐）

如果电脑安装了 Python，可以用命令行打开。

1. 解压项目压缩包。
2. 进入项目文件夹。
3. 在文件夹空白处按住 `Shift`，点击鼠标右键。
4. 选择“在终端中打开”或“在 PowerShell 中打开”。
5. 输入以下命令：

```bash
python -m http.server 8000
```

6. 打开浏览器，访问：

```text
http://localhost:8000/index.html
```

如果 `python` 命令无效，可以尝试：

```bash
py -m http.server 8000
```

如果浏览器询问是否允许摄像头，请选择“允许”，这样才能使用手势识别功能。

#### 方法二：使用 VS Code 或 Cursor 一键运行（Windows）

本项目可以通过 VS Code/Cursor 的调试配置实现一键运行：点击运行后，系统会自动启动本地服务器，并自动在浏览器中打开 `index.html`。

##### 1）准备条件

使用该方法前，请确认电脑已经安装：

- VS Code 或 Cursor。
- Python。
- Chrome 或 Edge 浏览器。

##### 2）创建 `.vscode` 配置文件夹

在项目根目录中新建一个名为 `.vscode` 的文件夹。

项目结构应类似：

```text
项目文件夹/
├─ index.html
├─ assets/
├─ 用户使用文档.md
└─ .vscode/
   ├─ launch.json
   └─ tasks.json
```

注意：`.vscode` 文件夹必须放在项目根目录，也就是和 `index.html` 同一级的位置。

##### 3）创建 `tasks.json`

在 `.vscode` 文件夹中新建 `tasks.json`，并写入以下内容：

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "start-local-server-8000",
      "type": "shell",
      "command": "python -m http.server 8000",
      "isBackground": true,
      "problemMatcher": []
    }
  ]
}
```

该文件的作用是：运行调试前，自动执行 `python -m http.server 8000`，在本地启动网页服务器。

##### 4）创建 `launch.json`

在 `.vscode` 文件夹中新建 `launch.json`，并写入以下内容：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Open index on localhost:8000",
      "type": "pwa-chrome",
      "request": "launch",
      "url": "http://localhost:8000/index.html",
      "webRoot": "${workspaceFolder}",
      "preLaunchTask": "start-local-server-8000"
    }
  ]
}
```

该文件的作用是：自动打开 Chrome，并访问 `http://localhost:8000/index.html`。

其中：

```json
"preLaunchTask": "start-local-server-8000"
```

表示在打开浏览器之前，先执行 `tasks.json` 中的 `start-local-server-8000` 任务。

##### 5）一键运行项目

配置完成后，按以下步骤运行：

1. 用 VS Code 或 Cursor 打开整个项目文件夹。
2. 点击左侧的“Run and Debug”。
3. 选择 `Open index on localhost:8000`。
4. 点击绿色运行按钮，或按 `F5`。
5. 系统会自动启动本地服务器，并在浏览器中打开网页。

如果浏览器询问是否允许摄像头，请选择“允许”，这样才能使用手势识别功能。

##### 6）如果 Python 命令无效

如果运行时提示 `python` 不是内部或外部命令，可以将 `tasks.json` 中的：

```json
"command": "python -m http.server 8000"
```

改为：

```json
"command": "py -m http.server 8000"
```

然后重新点击运行。

##### 7）如果 8000 端口被占用

如果终端提示端口被占用，或者运行任务出现 `exit code: 1`，通常说明电脑上已经有其他程序占用了 `8000` 端口。

可以将两个文件中的 `8000` 同时改成 `8080`。

`tasks.json` 中改为：

```json
"command": "python -m http.server 8080"
```

`launch.json` 中改为：

```json
"url": "http://localhost:8080/index.html"
```

注意：两个文件中的端口号必须保持一致。

#### 方法三：使用 Live Server 打开（Windows）

如果不想手动配置 `.vscode`，也可以使用 Live Server 插件打开。

1. 解压项目压缩包。
2. 打开 VS Code 或 Cursor。
3. 选择“打开文件夹”。
4. 选择解压后的项目文件夹。
5. 安装插件 `Live Server`。
6. 在左侧文件列表中找到 `index.html`。
7. 右键点击 `index.html`。
8. 选择 `Open with Live Server`。
9. 浏览器会自动打开网页。

如果浏览器询问是否允许摄像头，请选择“允许”，这样才能使用手势识别功能。

### Mac 用户打开方式

#### 方法一：使用 Python 本地服务器（Mac 推荐）

Mac 用户也建议通过本地服务器打开网页。Mac 中通常使用 `python3` 命令。

1. 解压项目压缩包。
2. 打开 Finder，进入解压后的项目文件夹。
3. 打开“终端 Terminal”。
4. 使用 `cd` 命令进入项目文件夹，例如：

```bash
cd /Users/你的用户名/Desktop/项目文件夹
```

5. 输入以下命令启动本地服务器：

```bash
python3 -m http.server 8000
```

6. 打开浏览器，访问：

```text
http://localhost:8000/index.html
```

如果 `python3` 命令无效，请先确认 Mac 是否已经安装 Python。可以输入：

```bash
python3 --version
```

如果能显示 Python 版本号，说明可以使用 `python3 -m http.server 8000` 启动本地服务器。

如果浏览器询问是否允许摄像头，请选择“允许”，这样才能使用手势识别功能。

可以直接双击 `index.html` 打开，但不推荐。

直接打开可能会出现以下问题：

- 预设 `.obj` 模型无法正常加载。
- 摄像头权限可能不可用。
- API 请求可能受到浏览器限制。
- 部分外部资源加载不稳定。

因此建议优先使用一键运行配置、Live Server 或 Python 本地服务器。

## 4. 基本功能说明

### 4.1 星云待机模式

打开网页后，系统会显示星云粒子待机效果。

点击左侧控制面板中的“切换到星云模式”，可以从文物展示状态返回星云状态。

### 4.2 导入 OBJ 文物

支持导入 `.obj` 格式的文物模型。

操作方式：

1. 点击“导入 OBJ 文件”。
2. 选择一个或多个 `.obj` 文件。
3. 导入后，文物会出现在文物展示列表中。
4. 点击列表中的文物名称即可切换展示。

也可以直接将 `.obj` 文件拖入网页进行导入。

### 4.3 文物展示列表

点击“当前对象”右侧的箭头按钮，可以打开文物展示列表。

列表功能：

- 点击文物名称：切换到该文物。
- 点击文物右侧的 `×`：删除该文物。
- 文物较多时：列表内部会自动滚动。
- 中文/英文切换时：列表名称会同步切换语言。

### 4.4 文物查看模式

点击“文物查看模式”按钮可以开启或关闭文物查看模式。

开启后：

- 鼠标拖拽可以移动观察视角。
- 鼠标滚轮可以对准光标位置进行缩放。

关闭后：

- 鼠标拖拽主要用于旋转文物或星云。
- 摄像机会回到默认展示位置。

### 4.5 粒子颜色切换

点击“粒子颜色”按钮，可以在三种颜色主题之间循环切换：

1. 绿色
2. 蓝色
3. 橙色

颜色切换会作用于：

- 星云粒子效果
- 文物粒子模型
- 当前正在展示的文物

### 4.6 中英文切换

点击 `Language: English` 或 `Language: 中文` 按钮，可以切换系统语言。

支持切换的内容包括：

- 功能介绍
- 按钮文字
- 状态文字
- 文物名称
- 文物描述
- 手势识别提示

### 4.7 手势识别交互

系统左下方或侧边区域会显示摄像头手势识别面板。

首次使用时，浏览器会请求摄像头权限，请选择“允许”。

支持手势：

- 手掌左右移动：控制星云或文物旋转。
- 拇指和食指张开：平滑缩放。
- OK 手势停留约 1.5 秒：切换到下一个文物。

如果摄像头无法使用，请检查：

- 浏览器是否允许摄像头权限。
- 是否通过 `localhost` 或 HTTPS 打开网页。
- 摄像头是否被其他软件占用。

## 5. 文物描述 API 功能

系统支持通过 API 自动生成文物描述，并支持中英文描述。

### 5.1 配置 API Key

打开 `index.html`，找到以下配置：

```javascript
const config = {
    count: 200000,
    colors: { dark: 0x000406 },
    apiKey: ""
};
```

将 `apiKey: ""` 改为你的 API Key：

```javascript
apiKey: "你的_API_KEY"
```

保存文件后重新打开网页。

### 5.2 描述生成方式

- 切换到某个文物后，右侧会显示文物描述面板。
- 系统会自动根据当前语言生成对应描述。
- 切换语言后，会自动显示或重新生成另一种语言的描述。
- 已生成的描述会被缓存，避免重复请求。

如果没有填写 API Key，系统会提示需要配置 API Key。

如果 API 请求失败，系统会显示默认兜底描述。

## 6. 常见问题

### Q1：网页打开后是空白怎么办？

请检查网络是否正常，因为系统需要加载 Three.js、MediaPipe 等外部库。

建议使用 Chrome、Edge 等现代浏览器。

### Q2：预设文物没有加载出来怎么办？

请确认：

- 是否通过本地服务器打开网页。
- `assets/models/` 文件夹是否存在。
- 模型路径是否没有被修改。
- `.obj` 文件名是否和代码中的预设路径一致。

### Q3：摄像头无法启动怎么办？

请确认：

- 浏览器是否允许摄像头权限。
- 是否使用 `http://localhost:8000`、Live Server 或 HTTPS 打开。
- 摄像头是否被其他软件占用。


### Q4：点击一键运行后没有打开网页怎么办？

请按顺序检查：

- `.vscode` 文件夹是否和 `index.html` 在同一级目录。
- `launch.json` 和 `tasks.json` 文件名是否拼写正确。
- `launch.json` 中的 `preLaunchTask` 是否和 `tasks.json` 中的 `label` 完全一致。
- 电脑是否已经安装 Python。
- `8000` 端口是否被其他程序占用。

如果端口被占用，可以把两个配置文件中的 `8000` 同时改成 `8080`。

### Q5：导入的文物没有英文名怎么办？

系统内置了常见三星堆文物词典，会尽量自动翻译中文文件名。

如果文件名非常特殊，系统会显示类似：

```text
Sanxingdui Artifact 1
```

建议将 `.obj` 文件命名为清晰的文物名称，例如：

```text
青铜神树.obj
青铜纵目面具.obj
金面罩.obj
```

### Q6：文物描述没有生成怎么办？

请检查：

- 是否已经配置 `apiKey`。
- 网络是否正常。
- API Key 是否有效。
- 浏览器控制台是否有请求错误。

## 8. 操作总结

常用操作如下：

- 导入文物：点击“导入 OBJ 文件”。
- 切换文物：打开文物列表并点击文物名称。
- 删除文物：点击文物列表中对应文物右侧的 `×`。
- 返回星云：点击“切换到星云模式”。
- 切换颜色：点击“粒子颜色”。
- 切换语言：点击“Language”。
- 开启查看模式：点击“文物查看模式”。
- 手势缩放：张开拇指和食指。
- 手势切换：OK 手势停留 1.5 秒。
